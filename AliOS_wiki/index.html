Kernel Entry Point (_start)

When GRUB loads AliOS 4, it follows the Multiboot2 protocol. The entry point is defined in assembly to bridge the gap between the bootloader and C code.

Assembly Header (boot.S)

section .text
bits 32 ; GRUB starts us in 32-bit Protected Mode

global _start
_start:
    cli             ; Disable interrupts
    mov esp, stack_top ; Setup stack

    ; Hand over to Long Mode Setup
    call check_multiboot
    call setup_paging
    call load_gdt
    
    jmp gdt64.code:long_mode_start

This code ensures the CPU is prepared for 64-bit instructions before jumping to kernel_main().

Global Descriptor Table (GDT)

In 64-bit Long Mode, segmentation is largely ignored, but a GDT is still required to define the code and data segments. AliOS 4 uses a "Flat Model."

The GDT Layout

Each entry is 8 bytes. The 64-bit Code Segment must have the L-bit (Long mode bit) set.

gdt64:
    dq 0 ; Null Descriptor (Required)
.code: ; Kernel Code Segment
    dq (1<<43) | (1<<44) | (1<<47) | (1<<53) ; Long Mode bit set
.data: ; Kernel Data Segment
    dq (1<<44) | (1<<47) | (1<<41)
.pointer:
    dw $ - gdt64 - 1
    dq gdt64
Technical Note: Even though base and limit are ignored in 64-bit mode, if the Code Segment isn't correctly defined, the CPU will Triple Fault the moment it tries to execute 64-bit C code.

PC Speaker Driver

Audio utilizes the PIT (Programmable Interval Timer) Channel 2.

Stuttering Update: The stuttering in AliOS 4 is confirmed to be related to IRQ timing. Without an IDT (Interrupt Descriptor Table) to handle precise timing, the square wave frequency "drifts" during heavy VGA writes.
