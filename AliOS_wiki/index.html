Kernel Entry Point (_start)

When GRUB loads AliOS 4, it follows the Multiboot2 protocol. The entry point is defined in assembly to bridge the gap between the bootloader and C code.

section .text
bits 32 ; GRUB starts us in 32-bit Protected Mode

global _start
_start:
    cli             ; Disable interrupts
    mov esp, stack_top ; Setup stack
    call setup_paging
    call load_gdt
    jmp gdt64.code:long_mode_start

Memory Manipulation & Debugging

AliOS 4 provides Ring 0 access to physical RAM. This is essential for debugging the Notebook UI and verifying the Kernel Heap state.

PEEK: Reading Raw Data

The peek command dereferences a pointer at a specific hex address. It allows the engineer to inspect the Interrupt Vector Table (IVT) or the BIOS Data Area.

POKE: Live Hardware Patching

By writing directly to 0xB8000, the operator can manipulate screen attributes in real-time. Usage of poke bypasses standard driver logic.

Security Warning: Overwriting addresses in the 0x100000 range will corrupt kernel machine code, resulting in an immediate General Protection Fault.

AliScript: The Native Programming Language

AliScript is a Stack-Oriented interpreted language built into aliscr.c. It allows for dynamic program execution within the shell.

Architecture: LIFO Stack

Instructions use "Post-Fix" notation. The stack is 64 slots deep, preventing overflow during complex math operations.

; Example: Math and Graphics
run 50 50 add print    ; Result: 100
run 0xB8000 0x41 poke  ; Writes 'A' to screen

VGA Notebook UI

The AliOS 4 UI operates in 80x25 Text Mode. Every character occupies 2 bytes: the ASCII character and the Attribute Byte.

Attribute 0x1E: Binary 0001 1110. Background: Blue (1), Foreground: Yellow (14).
